# Contributing

Thank you for contributing.

## Goals

Every contribution should improve:

- Accuracy
- Consistency
- Maintainability
- Reusability

---

## Skill Requirements

Each SKILL.md must include:

- Metadata
- Purpose
- When to Use
- Objectives
- Workflow
- Best Practices
- Common Pitfalls
- Output Format
- Validation Checklist
- Success Criteria

---

## Style Guide

Write using:

- Clear language
- Active voice
- Engineering terminology
- Actionable guidance

Avoid:

- Marketing language
- Ambiguous recommendations
- Tool-specific assumptions
- Unnecessary repetition

---

## Pull Request Checklist

- Structure follows template
- Examples included where appropriate
- Workflow complete
- Validation checklist present
- Output format specified
- Markdown lint passes

---

## Versioning

Breaking changes increment major version.

New capabilities increment minor version.

Documentation updates increment patch version.
